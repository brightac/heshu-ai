# Agent Review 内容三审工作流

AI 很快就能做出一份“看起来完成”的文章、视频稿或落地页。但看起来像成品，不等于可以发布。

Agent Review 用三个彼此独立的视角复核同一份内容。默认发布线是 **95 分**：任何一路低于 95，都必须给出可以直接替换的修改，再进入下一轮复审。

## 三个评审视角

1. **商业闭环评审**：检查受众、结果、产品承诺、价格、交付和行动入口是否连成一条线。
2. **传播表达评审**：检查标题、前 3 秒、数字密度、视觉动作、记忆点和品牌句是否足够有力。
3. **真实可信评审**：检查黑话、证据、统计口径、隐私、实际阻力，以及承诺与交付是否一致。

## 最小工作流

1. 用 [review-brief-template.md](./review-brief-template.md) 写清内容目标、产品事实和不可公开的信息。
2. 让三个评审独立阅读原稿，不先看彼此的意见。
3. 每个评审按 [scorecard-template.md](./scorecard-template.md) 打分，并提供可直接粘贴的替换文案。
4. 主 Agent 处理冲突，优先级固定为：**事实准确 > 承诺可兑现 > 传播效率**。
5. 修改后重新执行三路评审；全部达到 95 分，才允许发布。

## 可直接使用的总提示词

```text
请对这份内容执行 Agent Review。

建立三个彼此独立的评审视角：
1. 商业闭环：检查受众、结果、产品承诺、价格、交付和行动入口。
2. 传播表达：检查标题、前 3 秒、数字密度、画面动作、记忆点和品牌句。
3. 真实可信：检查黑话、证据、统计口径、隐私、实际阻力，以及承诺与交付是否一致。

三个评审不得先互相参考。每个评审必须输出：
- 0—100 分总分；
- 已成立的部分；
- 必须修改的问题；
- 每个问题对应的可直接替换文案；
- 缺少的证据或信息；
- “允许发布”或“不允许发布”。

主 Agent 汇总时，按“事实准确 > 承诺可兑现 > 传播效率”解决冲突。
修改后再次独立复审。只有三个分数都达到 95，才输出“允许发布”。
不得虚构数字、链接、用户证言或产品交付。
```

## 在贺叔 AI 产品里的位置

- **¥0 开源 Skills**：公开框架、提示词和模板，能自己跑通就不必付费。
- **¥199 / 年答疑群**：解决角色配置、标准冲突和落地问题。
- **¥1,999 实操工作坊**：现场把三审闸门装进自己的内容流水线。
- **一对一共创**：按业务风险定制评审标准，申请制。

它不是第四个价格档，也不承诺流量或收入。分数是发布纪律，不是客观真理；同一个 Agent 不能一边写、一边给自己放行。

公开前请先删除个人信息、客户数据、内部提示词和未获授权的证言。
